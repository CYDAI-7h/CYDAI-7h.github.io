# 《掷币者》交互设计方案

## 核心交互理念
创建一个沉浸式的概率圣殿体验，让用户通过互动仪式来理解概率论和金融哲学的深层含义。

## 主要交互组件

### 1. 掷币仪式模拟器
- **位置**: 主页面中央圣坛
- **功能**: 用户可以点击虚拟铜币进行投掷
- **机制**: 
  - 每次投掷记录结果（正/反）
  - 实时统计出现频率
  - 大数定理可视化：随着投掷次数增加，频率趋近于50%
  - 特殊事件：极低概率出现"立币"现象
- **视觉反馈**: 铜币3D旋转动画，结果用光效显示
- **音效**: 金属撞击声，低沉的诵经背景音

### 2. 概率圣殿漫游
- **位置**: 故事页面
- **功能**: 用户可以在圣殿中移动，探索不同区域
- **机制**:
  - 鼠标移动触发视差效果
  - 点击不同区域显示相关的哲学箴言
  - 三个朝见场景的沉浸式切换
- **视觉效果**: 光影变化，粒子效果，神秘氛围

### 3. 市场波动可视化
- **位置**: 哲学页面
- **功能**: 实时展示市场数据与概率规律的关系
- **机制**:
  - 模拟金融市场的随机游走
  - 用户可以调整参数观察结果变化
  - 展示均值回归现象
- **数据可视化**: 使用ECharts创建动态图表

### 4. 概率冥想室
- **位置**: 互动页面
- **功能**: 引导用户进行概率思维训练
- **机制**:
  - 随机事件预测游戏
  - 概率直觉测试
  - 风险决策模拟
- **反馈系统**: 显示用户的"概率感知准确度"

## 交互流程设计

### 用户旅程
1. **进入圣殿** → 被宏伟的建筑和神秘的氛围吸引
2. **初次掷币** → 体验随机性的魅力
3. **探索故事** → 通过三个朝见理解深层含义
4. **哲学思考** → 与市场数据互动，理解概率规律
5. **冥想练习** → 内化所学，获得智慧

### 情感曲线
- **敬畏** → **好奇** → **理解** → **感悟** → **升华**

## 技术实现要点

### 动画效果
- 使用Anime.js实现流畅的过渡动画
- 铜币投掷使用物理引擎模拟真实运动
- 粒子系统营造神秘氛围

### 数据可视化
- ECharts展示统计规律
- 实时数据更新和动画过渡
- 响应式设计适配不同屏幕

### 音效设计
- 环境音效营造沉浸感
- 交互音效提供即时反馈
- 音乐与叙事节奏同步

## 用户体验目标
通过交互式的概率探索，让用户不仅理解故事内容，更能亲身体验概率论的魅力，在互动中获得哲学启发。每个交互都应有明确的教育意义和情感价值。
